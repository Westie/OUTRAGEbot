CoreMaster::Format()

Description

Formats a string to mIRC-like colours and formatting.

List of current modifiers:
  • Formatting modifiers
    • {b} - Emboldens the string past that point.
    • {i} - Puts the text into italics.
    • {r} - Cancels all formatting and colours after that point.
    • {u} - Underlines the string past that point.
    • {v} - Inverts the background and foreground colours.
  • Colour modifiers - these follow the scheme of either:

    {c:foregroundColour}
    {c:foregroundColour:backgroundColour}

    These names are case insensitive:
    • White
    • Black
    • DarkBlue
    • DarkGreen
    • Red
    • Brown
    • Purple
    • Orange
    • Yellow
    • Green
    • Teal
    • LightBlue
    • Blue
    • Pink
    • DarkGrey
    • Grey

Prototype

boolean CoreMaster::Format(string $sInputString)

Example and Output

$this->Format("{b}This string is in bold and {c:yellow}this text is yellow{r}, but this section has no custom formatting.");
<strong>This string is in bold and <span style="color:yellow">this text is yellow</span></strong>, but this section has no custom formatting.

CoreMaster::Raw()

Description

Sends a raw message to the IRC server.

List of possible types for $mOption:
  • Output options
    • SEND_DEF - option defined in configuration file
    • SEND_MAST - sent from the first defined IRC slave (in this case, the master)
    • SEND_CURR - sent from the current IRC child
    • SEND_ALL - sent from all at once
    • SEND_DIST - each message is distributed between each bot.
  • Output modification
    • FORMAT - formats the string in accordance with CoreMaster::Format()

Setting the $mOption argument null, will result in the second argument becoming SEND_DIST.

CoreMaster::addEventHandler()Event handlers

Description

One of the uses for addEventHandler is to listen into a built in event. You can do this by creating an event handler with the pre-registered callbacks, such as onChannelMessage().

Please note: If the callback isn't a member of any class that is a child of Script, then there will be an argument prepended that acts as the replacement for $this. This means that closures and global scope functions will have that extra argument.

Prototype

string CoreMaster::addEventHandler(string $sEvent, callback $cCallback)

Example

class EventExample extends Script
{
        /**
         *      Called when the script is constructed.
         */
        public function onConstruct()
        {
                # This will require the CoreMaster instance as the first argument.
                $this->addEventHandler("onChannelMessage", function($irc, $channel, $user, $message)
                {
                        $irc->Message($channel, "{$user} said something, therefore he just got nick alerted.");
                });
                
                
                # This example doesn't need the prepended argument, however.
                $this->addEventHandler("onChannelMessage", "Hello");
        }
        
        
        /**
         *      Called when someone speaks in the channel.
         */
        public function Hello($channel, $user, $message)
        {
                if(preg_match("/^hello$/", $m))
                {
                        $c("hello world!");
                }
        }
}
CoreMaster::addEventHandler()Numeric handlers

Description

One of the uses for addEventHandler is to listen for IRC numerics and commands. You can do this by creating an event handler, listening for perhaps, invite requests. The format codes, mentioned below, can be placed in any order, dependent on how they're needed.

List of format codes

  • c - Message parts as an array.
  • m - Message object. (Core::getMessageObject)
  • p - The payload, or rather the bit after the semi-colon.
  • r - The raw, unparsed input string.
  • h - The parsed hostname, as an object.
  • u - User's nickname, as a string.

Please note: If the callback isn't a member of any class that is a child of Script, then there will be an argument prepended that acts as the replacement for $this. This means that closures and global scope functions will have that extra argument.

Prototype

string CoreMaster::addEventHandler(string $sNumerics, callback $cCallback [, string $sArgumentFormat = "m" ])

Example

class NumericExample extends Script
{
